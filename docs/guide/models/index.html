Models

Introduction to Models​

In VisionCloud, a Model represents a deep learning model that you can train to perform specific computer vision tasks. Models are the intelligent core of your computer vision solutions, capable of learning from your labeled datasets to make predictions on new, unseen images.

Each model is designed for a specific type of task (classification, detection, segmentation, etc.) and can be configured, trained, and optimized to meet your unique requirements. Once trained, models can be deployed to production environments to automate visual inspection, analysis, or decision-making processes.

Key Concepts​

  • Model: A configurable deep learning model that can be trained on your data
  • Model Type: The specific task architecture (classification, segmentation, etc.)
  • Version: Each training run creates a new version of your model
  • Parameters: Configuration settings that control the model's architecture and training process
  • Backbone: The pre-trained neural network that forms the foundation of your model

Think of models as specialized employees you can train to perform specific visual tasks with increasing accuracy over time.

Create Model​

Inside any project, heading to the Models section opens up a powerful control center.
Here, two main actions become available:

  1. Create new Deep Learning models from scratch.
  2. Browse and inspect the details of models you've already created.

This is the starting point for managing all your AI training workflows in one place.

The models list view showing available models in your project

Create Deep Learning Model​

Clicking to create a model brings up a simple and intuitive pop-up.
Here you can:

  1. Give your model a unique name.
  2. Select the type of model best suited to your task.

When naming your model, consider using a convention that includes:

  • The purpose of the model (e.g., "ProductDefectDetector")
  • The dataset it uses (e.g., "MachinePartsInspector")
  • A version indicator (e.g., "FabricQualityV1")

Clear naming helps when you need to manage multiple models or versions for different purposes.

Creating a new model with a descriptive name and appropriate type

Available Model Types​

Each model type is optimized for a different kind of computer vision task:

Image Classification Models​

  • Generic classification: Assigns one or more labels to the entire image

    • Use for: Product categorization, quality grading, scene recognition
    • Example applications: Sorting products by type, identifying defective vs. normal items
    • Input: Whole images
    • Output: Class labels with confidence scores

  • Dynamic classification: Adds flexibility, supporting changing or context-based labels

    • Use for: Applications where categories may evolve over time
    • Example applications: Detecting new product variants, adapting to seasonal changes
    • Input: Whole images
    • Output: Adaptable class labels with confidence scores

  • Generic classification embedding: Turns images into vector representations

    • Use for: Similarity searching, clustering, anomaly detection
    • Example applications: Finding visually similar products, identifying unusual items
    • Input: Whole images
    • Output: Feature vectors that capture image characteristics

Segmentation Models​

  • Generic segmentation: Breaks the image into meaningful regions based on visual features
    • Use for: Precise object outlining, area measurement, detailed analysis
    • Example applications: Measuring defect areas, analyzing medical images
    • Input: Whole images
    • Output: Pixel-level masks indicating region classifications
  • Instance segmentation (OLD): Differentiates between multiple objects of the same type
    • Use for: Counting and analyzing multiple objects with precise boundaries
    • Example applications: Analyzing cells in microscopy, identifying multiple defects on a surface
    • Input: Whole images
    • Output: Individual masks for each object instance with class labels

Localization Models​

  • Segmentation localization: Highlights and localizes specific parts of an image
    • Use for: Finding specific regions within larger objects
    • Example applications: Identifying components on circuit boards, locating specific anatomical features
    • Input: Whole images
    • Output: Localized areas with segmentation masks

Choosing the Right Model Type​

Selecting the appropriate model type is critical for achieving optimal results:

  1. Consider your task requirements:

    • Need to assign categories to entire images? → Classification
    • Need to locate objects within images? → Detection
    • Need precise object boundaries? → Segmentation
    • Need to find exact points? → Keypoint detection

  2. Consider your data characteristics:

    • Single object per image → Simple classification may suffice
    • Multiple objects of same class → Instance segmentation or detection
    • Need to measure areas precisely → Segmentation is required
    • Finding similarities between images → Classification embedding

  3. Consider computational constraints:

    • Classification models are typically faster and lighter
    • Segmentation models require more processing power
    • Detection models offer a balance of speed and precision

💡 Pro Tip: If you're unsure which model type to choose, start with the simplest option that might solve your problem. You can always create additional models with different architectures to compare performance.

Overview Interface​

Opening a model gives access to its status and configuration in a clean sidebar layout.

The model's current state is clearly shown (Training, Error, or OK), and multiple sections organize all the information and tools you need.

Sections Breakdown​

  1. Definition – Set up the input format, patch configuration, and define the model outputs.
  2. Datasets – Manage training and validation datasets used by the model.
  3. Parameters – Adjust architecture, data, and training settings.
  4. Progress – See real-time updates on training performance.
  5. Model Status – View the model's current status (idle, training, error, or completed).

This layout allows you to track, modify, and manage your model with precision.

Top Toolbar​

In the top-right corner of the model interface, a compact toolbar offers key controls:

  1. Switch model versions – Easily navigate between different trained versions.
  2. Download the model – Export the model for inference or backup.
  3. Start training – Kick off training when you're ready.

These tools ensure that experimentation and deployment are always just a click away.

Definition Section​

This section lays the groundwork for how the model processes data.

  • Input Configuration: Define image dimensions and channels (RGB or grayscale).
    ⚠️ Note: Images larger than 700 x 700 px may cause training errors. In this case, use Patch Mode to divide images into smaller sub-images. It's highly recommended to apply an overlap when patching to avoid cutting important content.
  • Region of Interest (ROI): Select a specific area within each image to focus the training.
  • Patch Mode: Automatically or manually split large images into smaller patches.
  • Output Configuration: Define output names (special characters are not allowed) and structure your model's results.

This configuration ensures that your model receives well-prepared data and that outputs are clearly defined.

Datasets​

In the Datasets section, data is brought into the training pipeline.

You can:

  1. Attach a new dataset from your project.
  2. Edit or remove datasets that are already linked to the model.

🔔 Note: To start training, at least one training dataset and one validation dataset must be attached.
It's highly recommended that the validation set represents around 20% of the total images.

Add Dataset​

When adding a dataset, a helpful dialog lets you:

  1. Select an existing dataset from the project.
  2. Choose whether it's used for training, validation or test.
  3. Map each label in the dataset to the model's outputs as defined in the Definition section.
  4. Select the weight of the dataset in the training process.

Parameters​

The Parameters section is where you configure the deep learning architecture and training process for your model. Making the right parameter choices can significantly impact training time, model accuracy, and generalization ability.

Parameters are grouped into three main categories to help you navigate the many options available:

  • Model Architecture: Define the neural network structure and behavior
  • Dataset Parameters: Control how your data is processed and augmented
  • Training Parameters: Configure the optimization process

Each parameter category affects different aspects of your model's performance and behavior.

Parameters section with the three main configuration categories

Model Architecture Parameters​

The architecture parameters define the fundamental structure of your neural network, affecting its capacity, speed, and ability to learn different features.

Backbone Selection​

The backbone is the pre-trained neural network that serves as the foundation for your model. Different backbones offer different tradeoffs between accuracy and computational efficiency:

Backbone FamilyCharacteristicsBest For
SimpleSimple and fast, but limited in capacityBasic classification tasks
ResNet-SimpleCustom implemntation of ResNetComplex classification tasks
ResNetGood feature extraction, residual connections prevent vanishing gradientsGeneral purpose, excellent starting point for most tasks
VGGSimple architecture, strong feature extractionTasks requiring fine-grained visual features
DenseNetDense connections, parameter efficientComplex visual tasks with limited training data
EfficientNetOptimized for efficiency, scales well across different sizesMobile applications, resource-constrained environments
DensenetDense connections, parameter efficientComplex visual tasks with limited training data

How to choose:

  • For your first model, start with ResNet50 - it offers a good balance of accuracy and speed
  • If your device has limited resources, try MobileNet or EfficientNetB0
  • For complex tasks with lots of training data, consider ResNet101 or larger
  • If speed is critical, use MobileNet or EfficientNetB0

💡 Pro Tip: For most industrial applications, ResNet50 provides an excellent balance of accuracy and performance. Only switch to more complex backbones if you have sufficient data and computational resources.

Loss Function​

The loss function measures how far your model's predictions are from the true values, guiding the learning process:

Classification Loss Options:

  • Cross Entropy: Standard loss for classification, works well for balanced classes
  • Focal Loss: Focuses on hard examples, useful when classes are imbalanced
  • Binary Cross Entropy: Used for multi-label classification where multiple labels can apply

Segmentation Loss Options:

  • Dice Loss: Measures overlap between predicted and true masks, good for imbalanced pixel classes
  • IoU Loss: Based on Intersection over Union, robust to class imbalance
  • Cross Entropy: Can be used for segmentation but may struggle with class imbalance

Detection Loss Options:

  • Combination of regression losses (for bounding box coordinates) and classification losses (for object classes)

How to choose:

  • For balanced classification tasks: Cross Entropy
  • For imbalanced classes (some classes appear much more frequently): Focal Loss
  • For segmentation with varying sized objects: Dice Loss or IoU Loss

Feature Dimensions​

This parameter controls the size of feature maps in your network:

  • Higher dimensions (256, 512): More capacity to learn complex patterns, but require more memory
  • Lower dimensions (64, 128): More efficient, but may limit model capacity

How to choose:

  • For simple visual tasks (basic classification): 64-128 dimensions
  • For complex tasks (detailed segmentation): 256-512 dimensions
  • If you see underfitting (high training and validation loss): Try increasing dimensions
  • If you experience out-of-memory errors: Decrease dimensions

Dataset Parameters​

Dataset parameters control how your images are processed before and during training, affecting what your model "sees" during the learning process.

Data Preprocessing​

These settings determine how images are prepared before feeding them to the model:

  • Normalization: Standardizes pixel values to help training converge faster

    • Always keep enabled for consistent model performance
    • Uses dataset mean and standard deviation or preset values

  • Resizing Strategy: Controls how images are resized to match input dimensions

    • Stretch: Stretches image to fit (may distort aspect ratio)
    • Fit: Preserves aspect ratio, adds padding if needed (recommended for most cases)
    • Crop: Crops image to fit (may lose information at edges)

Best practice: Use "Fit" strategy for most applications to preserve visual integrity while ensuring consistent input size.

Data Augmentation​

Augmentation artificially expands your training data by applying transformations to create variations of your images. This helps your model generalize better to new, unseen images.

Common augmentations include:

AugmentationEffectWhen to UseWhen to Avoid
Horizontal FlipMirrors image horizontallyMost applicationsWhen left/right orientation matters (text, asymmetric objects)
Vertical FlipMirrors image verticallySatellite imagery, microscopyWhen up/down orientation matters (most real-world objects)
RotationRotates image by random anglesObjects that can appear at any orientationWhen orientation is fixed in your application
Brightness/ContrastAdjusts lighting conditionsOutdoor applications or varying lightingHigh-precision color-based analysis
Zoom/ScaleSimulates objects at different distancesObject detection with size variationWhen scale is fixed in your application
ShearApplies angular distortionAdding variation for more robust modelsPrecision measurement applications
Color JitterVaries hue, saturation, etc.Natural scene analysisMedical imaging, color-critical applications

Recommended configurations:

  • For general classification: Enable horizontal flip, minor rotation (±15°), brightness/contrast adjustments
  • For object detection: All of the above plus minor scaling/zoom variations
  • For precise segmentation: Limited augmentations - horizontal flip and subtle brightness adjustments
  • For outdoor scenes: More aggressive brightness/contrast and weather augmentations

⚠️ Important: Choose augmentations that reflect realistic variations your model will encounter in production. Unrealistic augmentations can harm performance.

Patch Settings​

For very large images (e.g., high-resolution scans, satellite imagery), patch mode divides them into smaller sections for processing:

  • Patch Size: Dimensions of each sub-image (e.g., 512×512 pixels)
  • Overlap: How much patches should overlap (e.g., 25%)
    • Higher overlap (30-50%) ensures objects aren't cut between patches
    • Lower overlap (10-20%) reduces computation but may miss objects at boundaries

When to use patching:

  • Images larger than 1000×1000 pixels
  • High-resolution medical or industrial imagery
  • Satellite or aerial photography
  • When objects of interest are small relative to the full image

Recommended settings:

  • For large objects: Larger patches (800×800) with 15-20% overlap
  • For small objects: Smaller patches (512×512) with 30-50% overlap

Training Parameters​

These settings control how your model learns from data during the training process.

Optimization Settings​

  • Learning Rate: Controls how quickly the model learns

    • Too high: Training becomes unstable or diverges
    • Too low: Training is very slow and may get stuck
    • Typical values: 0.1 (SGD), 0.001 (Adam), 0.0001 (fine-tuning)

    Effects of different learning rates on training

  • Weight Decay: Prevents overfitting by penalizing large weights

    • Higher values (0.001-0.01): More regularization, good for small datasets
    • Lower values (0.0001): Less regularization, good for large datasets

Training Schedule​

  • Batch Size: Number of images processed together

    • Larger batches (16-32): More stable gradients, faster training
    • Smaller batches (4-8): Use when memory is limited
    • Recommended: Use the largest batch size that fits in your GPU memory

  • Epochs: Maximum number of complete passes through the training dataset

    • Too few: Underfitting, model doesn't learn enough
    • Too many: Potential overfitting, wasted computation
    • Typical range: 50-200 epochs
    • Best practice: Set a higher number and rely on early stopping

  • Early Stopping: Automatically stops training when no improvement is seen

    • Patience: Number of epochs to wait for improvement (typically 5-15)
    • Metric: Which performance measure to monitor (loss, accuracy, etc.)
    • Mode: Whether to monitor for increasing or decreasing metric

    Early stopping prevents overfitting by halting training when validation metrics plateau

Additional Training Settings​

  • Class Balancing: Adjusts for imbalanced datasets
    • When to use: When some classes have many more examples than others
    • How it helps: Prevents model from ignoring minority classes
  • Mixed Precision: Uses lower precision calculations to speed up training
    • Benefit: Faster training, lower memory usage
    • Trade-off: Slightly reduced numerical precision
    • Recommendation: Keep enabled unless you notice instability

Progress​

The Progress section provides real-time feedback on your model's learning process.

You'll find:

  1. A real-time loss graph showing how well the model is performing.
  2. A panel at the bottom-left of the graph area that allows you to customize the display and scale.

This view is essential for tracking training evolution and quickly spotting any issues.

Learning Metrics Visualization​

The progress visualization includes several key metrics to help you monitor and evaluate your model's training:

  • Loss curves: Track the decrease in training and validation loss over time
  • Learning rate: View the automatic learning rate adjustments during training

Troubleshooting & Common Issues​

When working with deep learning models, you may encounter certain challenges. Here are solutions to common issues:

Training Issues​

  • Loss not decreasing:

    • Check learning rate (may be too high or too low)
    • Verify dataset correctness and label quality
    • Try a different model architecture
    • Check for class imbalance issues

  • GPU out of memory:

    • Reduce batch size
    • Use smaller input image dimensions
    • Choose a lighter backbone architecture
    • Enable mixed precision training

  • Overfitting:

    • Add more training data
    • Increase data augmentation
    • Add regularization (dropout, weight decay)
    • Simplify model architecture

Export/Deployment Issues​

  • Export failures:

    • Check for unsupported operations in target format
    • Verify hardware compatibility
    • Ensure consistent input/output specifications
    • Check for size limitations in the target platform

  • Slow inference:

    • Optimize batch size for target hardware
    • Try quantization to lower precision
    • Use hardware acceleration where available
    • Consider model pruning techniques

  • Inconsistent results:

    • Verify preprocessing matches training pipeline
    • Check normalization values
    • Ensure consistent resolution handling
    • Verify color space handling (RGB/BGR)
