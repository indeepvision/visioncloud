Projects

Creating a Project​

To get started, go to the Projects page and click on the Create Project button.

You'll be prompted to fill out the necessary information such as:

  • Project name: A clear, descriptive name for your project
  • Description: Optional details about the project's purpose, goals, or contents

Create new project

Once your project is created, it will appear in the list, and you can access it anytime from the Projects section.

Projects Page​

The Projects dashboard provides an overview of all your available projects with key information:

  • Project name and description
  • Creation date
  • Last edition date
  • Number of elements and size
  • Description

Project Filtering and Sorting​

Easily find specific projects using the available tools:

  • Search bar: Find projects by name or description
  • Filters: Filter by status, date, or other attributes
  • Sort options: Order projects by name, date, or other criteria


Projects dashboard with filtering options

Project Management​

Project Actions​

Each project has several management options available through the project actions menu:

  • Edit: Modify project name, description, or other details
  • Duplicate: Create a copy of the project with all its contents
  • Delete: Permanently remove the project (requires confirmation)

Project Permissions​

VisionCloud uses a resource-based access control system for projects. Permissions are assigned per project for different resources (datasets, models).

Each resource has the following permission levels:

  • None: No access to the resource
  • Viewer: Can see resource listings but cannot access details
  • User: Can view resources and their details but cannot modify them
  • Editor: Full access to create, view, modify, and delete resources

In addition, at the project level:

  • Admin: Full control over the project, including ability to manage users and their permissions
  • Owner: Highest level of access, with ability to transfer ownership

To manage permissions:

  1. Go to the Settings page of the organization
  2. View current users and their permission levels
  3. Select a specific user
  4. Attach the user to the project with the desired permissions

Only project Admins and Organization Admins can modify project permissions.

Inside a Project​

When you open a project, you'll find two main areas:

  • Datasets – Collections of labeled or unlabeled images for training
  • Models – Your trained neural network models

Project Best Practices​

Maximize the effectiveness of your projects with these organizational practices:

Naming Conventions​

  • Use clear, descriptive names that indicate the project's purpose
  • Include version information if appropriate (e.g., "ProductDefects_v2")
  • Consider adding prefixes to group related projects (e.g., "QC", "RnD")

Project Structure​

  • Keep related datasets and models within the same project
  • Use separate projects for different product lines or significantly different use cases
  • Consider creating dedicated test projects to experiment with new approaches

Documentation​

  • Provide detailed project descriptions explaining the purpose and goals
  • Document key decisions and approach in the project description
  • Keep track of important performance metrics and milestones

Collaboration​

  • Assign clear roles based on team members' responsibilities
  • Use viewer access for stakeholders who need to monitor but not modify
  • Regularly review and update permissions as team compositions change
  • Schedule regular reviews of project status with all stakeholders
